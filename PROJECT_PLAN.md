# План проекта Event Kvartal

## 1. Разбор действующего сайта

Сделано:
- Сохранены страницы `/`, `/services`, `/events`, `/contacts`, `/aboutus`, юридические страницы.
- Зафиксировано, что `/cases` и `/posters` сейчас отдают 404.
- Скачаны 100 медиафайлов из Tilda.
- Вытащены ключевые блоки: услуги, направления, команда, контакты, юридические ссылки.

Нужно ещё:
- Выбрать финальные фото для публичной версии.
- Проверить, какие кейсы и афиши реально нужны, потому что на текущем сайте эти разделы не работают.
- Уточнить название: публично оставить «Спутник» или переходить на «Event-квартал».

## 2. Новый визуал

Направление:
- Взрослый event-сайт: светлый фон, графитовый текст, оранжевый акцент бренда.
- Первый экран сразу объясняет, что компания делает: event, art, travel, loft, service.
- Фото использовать как доказательство опыта, не как тёмный декоративный фон.
- Убрать ощущение «детского» сайта: меньше мультяшных форм, меньше случайных скруглений, больше сетки и ясной типографики.

Перед финальным редизайном:
- Попросить у Андрея 2-3 референса по стилю.
- Отдельно согласовать, нужен ли строгий B2B-вид или более праздничный B2C.

## 3. Структура сайта

Стартовая версия:
- Главная: hero, направления, услуги, процесс, команда, контакты, заявка.
- Услуги: корпоративы, MICE, арт-мастерские, travel, техническое оснащение, сервис.
- Мероприятия: event / art / travel и примеры форматов.
- О нас: опыт, команда, подход.
- Контакты: адрес, телефон, почта, реквизиты.

Следующий этап:
- Добавить отдельные страницы услуг.
- Добавить кейсы вместо неработающего `/cases`.
- Добавить афиши, если клиент реально ведёт календарь событий.

## 4. Админка

Стартовая версия:
- Логин/логаут.
- Редактирование JSON-контента главной страницы.
- Просмотр заявок.

Дальше:
- Нормальные формы вместо JSON: услуги, команда, контакты, кейсы.
- Загрузка фото через админку.
- Статусы заявок и экспорт в CSV.

## 5. Технический план

Стек:
- PHP 8.x.
- SQLite в `data/event_kvartal.db`.
- Без сборки, без npm, чтобы легко положить на Beget.

Перед деплоем:
- Сменить пароль админки.
- Закрыть `_source/` от публикации или не копировать на прод.
- Проверить `pdo_sqlite` на хостинге.
- Добавить `.htaccess`, если нужны красивые URL без `.php`.
